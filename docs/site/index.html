The training loop, collapsed.

Today, improving a deployed model looks like: collect failure examples, format them, upload to a training service, wait hours, download new weights, redeploy, hope. Kiln collapses that into one API call. 

# Submit a correction — the model learns it in seconds
curl http://localhost:8420/v1/train/sft \
  -H "Content-Type: application/json" \
  -d '{
    "examples": [
      {"messages": [
        {"role": "user", "content": "Summarize this contract clause..."},
        {"role": "assistant", "content": "The clause establishes..."}
      ]}
    ]
  }'

# The next request already uses the updated weights
curl http://localhost:8420/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "Summarize this contract clause..."}]}'

A 4B model continuously tuned to your specific workload will outperform a generic 70B model on the tasks you actually care about — on hardware you already own. Kiln targets a single model (Qwen3.5-4B) and tunes the scheduler, memory manager, and kernels for it. This isn't a general-purpose framework. It's a scalpel.

What you get

Continuous batching & paged KV cache

Sarathi-style chunked prefill keeps decode requests from stalling behind long prompts. Virtual-memory KV blocks eliminate fragmentation. 128K context fits in ~13 GB on 24 GB GPUs.

Live LoRA training over HTTP

POST /v1/train/sft with chat examples or POST /v1/train/grpo with scored completions. Runs in-process on the loaded model — no second copy in VRAM, no Python sidecar.

Hot-swap adapters in seconds

New adapter weights activate atomically at iteration boundaries. Zero downtime. Load, unload, merge (TIES, weighted-average, concat), upload, download, compose — full adapter lifecycle.

Single Rust binary, single GPU

One process, one model in memory, one binary to deploy. RTX 3090 / 4090 / A6000 on the CUDA side; M-series Macs on Metal. OpenAI-compatible API with SSE streaming.

The GRPO loop

Generate completions, score them with your own reward function, feed the results back. The model learns what “good” means for your use case. Your reward function can be anything — regex, unit tests, another model, human eval. 

# 1. Generate candidates
responses = [client.chat.completions.create(
    model="qwen3.5-4b-kiln", messages=[...], temperature=0.7
) for _ in range(8)]

# 2. Score them however you want
scored = [{"text": r.choices[0].message.content, "reward": my_score(r)}
          for r in responses]

# 3. Submit — server trains and hot-swaps immediately
requests.post("http://localhost:8420/v1/train/grpo",
              json={"groups": [{
                  "messages": [{"role": "user", "content": prompt}],
                  "completions": scored,
              }]})

# 4. Next inference already uses the improved weights

See the GRPO guide for worked verifiable-rewards examples (math, JSON, code).

Install

Latest release: kiln-v0.2.13. Prerequisites: NVIDIA GPU with 24 GB+ VRAM and CUDA 12.4 or Apple Silicon Mac with 16 GB+ unified memory.

Desktop App · recommended quick path

Prefer the GUI? Download Kiln Desktop desktop-v0.2.2. On first launch, the Desktop App downloads and verifies the matching kiln server binary for you. No Rust toolchain, CUDA toolkit, or Xcode install is required for the GUI path.

Use the server binary commands below when you want to run kiln serve directly or automate a headless host.

Linux x86_64 · CUDA 12.4

curl -L -o kiln.tar.gz \
  https://github.com/ericflo/kiln/releases/download/kiln-v0.2.13/kiln-0.2.13-x86_64-unknown-linux-gnu-cuda124.tar.gz
tar -xzf kiln.tar.gz
KILN_MODEL_PATH=./Qwen3.5-4B ./kiln serve

macOS · Apple Silicon (Metal)

curl -L -o kiln.tar.gz \
  https://github.com/ericflo/kiln/releases/download/kiln-v0.2.13/kiln-0.2.13-aarch64-apple-darwin-metal.tar.gz
tar -xzf kiln.tar.gz
KILN_MODEL_PATH=./Qwen3.5-4B ./kiln serve

Windows x86_64 · CUDA 12.4

Invoke-WebRequest -Uri `
  https://github.com/ericflo/kiln/releases/download/kiln-v0.2.13/kiln-0.2.13-x86_64-pc-windows-msvc-cuda124.zip `
  -OutFile kiln.zip
Expand-Archive kiln.zip
$env:KILN_MODEL_PATH = ".\Qwen3.5-4B"; .\kiln\kiln.exe serve

Docker (Linux + NVIDIA Container Toolkit)

docker pull ghcr.io/ericflo/kiln-server:latest
docker run --gpus all -p 8420:8420 \
  -v /path/to/Qwen3.5-4B:/models \
  ghcr.io/ericflo/kiln-server:latest serve

Each release is signed and SHA-256 checksums are published as .sha256 sidecars on the releases page. The Quickstart walks through downloading model weights, the first chat completion, the first SFT POST, and the first GRPO loop.
