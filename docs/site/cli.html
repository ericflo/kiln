CLI Reference · serve · train · adapters

Run Kiln from the terminal.

Use the CLI when you want a headless server, repeatable scripts, or quick diagnostics against a running Kiln instance. If you want visual status, adapter controls, or training monitoring, open http://127.0.0.1:8420/ui instead.

Server

Start serving Qwen3.5-4B

Point KILN_MODEL_PATH at the local model directory, then start the OpenAI-compatible server. Running kiln with no subcommand also serves.

export KILN_MODEL_PATH=./Qwen3.5-4B
kiln serve

The default server listens on 127.0.0.1:8420; open /ui there for the dashboard.

Configuration

Use config files and model IDs

--config loads a TOML config file. --served-model-id changes the model name returned by /v1/models and accepted by OpenAI-compatible clients.

kiln serve --config kiln.toml
kiln serve --served-model-id qwen3.5-4b-local
kiln config --file kiln.toml

Use kiln config --file to validate a config before starting the server.

Health

Check server readiness

kiln health prints a readable tree with model, adapter, scheduler, and training status.

kiln health
kiln health --url http://localhost:8420

Use --json in scripts or CI probes when you want the raw health payload.

kiln health --json \
  | python3 -m json.tool

Training

Submit SFT and GRPO jobs

Training commands talk to an already-running server. SFT reads JSONL correction examples; GRPO reads one JSON request or batch with prompts, completions, and rewards.

SFT corrections

kiln train sft \
  --file corrections.jsonl \
  --adapter support-bot

Add --epochs, --lr, or --lora-rank when you need to override defaults.

GRPO rewards

kiln train grpo \
  --file grpo-batch.json \
  --adapter support-bot

Use this for generate → score → train loops with verifiable rewards.

Queue status

kiln train status
kiln train status --job-id train_123

Use the dashboard when you want visual progress and recent job history.

Adapters

Manage LoRA adapters

Adapter commands call the running server's adapter API. Use them in scripts; use /ui when you want upload, download, merge, or safer visual confirmation before deleting.

List, load, unload

kiln adapters list
kiln adapters load support-bot
kiln adapters unload
kiln adapters unload support-bot

The named unload form is accepted for backwards compatibility; the server unloads the active adapter.

Delete

kiln adapters delete support-bot

Delete removes an adapter through the server. Prefer the UI for one-off manual cleanup.
